## Technical Architecture

See the live application specification generated by the `generate-spec` edge function for full technical details: `src/lib/masterPromptContent.ts` (cached) or call `/functions/v1/generate-spec` (live).

### Key Design Principles

1. **City-Agnostic Pipeline** — All content generation is parameterized by city; no hardcoded city logic in the core pipeline.
2. **Zero-Touch Automation** — Cron-driven scheduling with fail-open behavior; posts publish even if non-critical steps fail.
3. **Learning Feedback Loop** — `analyze-performance` and `analyze-growth` feed winning patterns back into `generate-caption`.
4. **Visual Learning Firewall** — Degraded fallback renders (`gradient_only`, `degraded_fallback`) are excluded from the learning set so the AI does not learn to prefer failure.
5. **Tiered Fallback Chain** — Creatomate → JSON2Video → Ken-Burns static; video-required platforms (YouTube, TikTok, Instagram Reels) never fall back to static images.

## Development Methodology

This project follows the **Investigation-First Development Protocol (IFDP)**:

1. **Phase A: Investigation** — Gather evidence, identify root cause, propose fixes. No code changes.
2. **Phase B: Implementation** — Implement approved fixes, add tests, deploy.
3. **Phase C: Verification** — Monitor 24–48h, compare before/after metrics.

See `docs/IFDP_METHODOLOGY.md` for full details.

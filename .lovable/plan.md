## 1. Slot-based title + location beacon (publish layer)

In `process-scheduled-posts/index.ts` → `buildHookTitle(...)`:

- Accept an optional `slot` arg (`"morning" | "afternoon" | "evening" | "adhoc"`), passed from the caller that already knows the slot (the same `post.caption` `[auto:slot]` regex used at line ~1685, or `preflight.slot`).
- Replace the current `[${stamp}]` prefix with a slot-aware prefix:
  - morning → `[8 AM]`
  - afternoon → `[1 PM]`
  - evening → `[6 PM]`
  - fallback (adhoc/unknown) → keep existing `getCityLocalStamp(city)` behavior so nothing regresses.
- Keep the 95-char cap and ellipsis logic intact.

Location beacon in caption:
- After the AI caption is produced, ensure the first line is `📍 ${city} · ${SlotName} Update` (Morning/Afternoon/Evening/Update). The existing `getCityLocalStamp` injection in `generate-caption/index.ts` already prepends a 📍 line — extend that block to use slot name when slot is known (passed via request body), otherwise keep current time-stamp text. No new line is added if one is already present; it is replaced (same pattern used today).

## 2. Anti-clone / hook diversity (generation layer)

In `generate-caption/index.ts` (and the mirrored `SKYBRIEF_SYSTEM_PROMPT` in `process-scheduled-posts/index.ts`), append a new block to the system prompt:

- **Personality rotation by slot** (slot passed in request body, default "afternoon"):
  - morning → Upbeat / Energetic
  - afternoon → Informative / Direct
  - evening → Calm / Cinematic
- **Anti-repeat rule**: "Do NOT reuse the opening hook, first sentence structure, or CTA verb from the previous post for this city. Previous opener: `<prev_opener>`. Previous CTA: `<prev_cta>`."
- **CTA pool of 4**, rotated deterministically by `(date + slot)` index:
  1. "Tap ❤️ if this helped your plans."
  2. "Subscribe for tomorrow's brief."
  3. "Check the radar before you head out."
  4. "Drop your city's weather in the comments."
- Inject `<prev_opener>` and `<prev_cta>` by fetching the most recent `post_history` row for this `user_id` + `city` (limit 1, ordered by `created_at desc`). Best-effort; if none, omit those lines.

## 3. Similarity validation (safety)

After the AI returns the caption, compute a normalized similarity (lowercased, punctuation-stripped, token-set Jaccard) against the previous `post_history.caption` for the same city. If `>= 0.80`:

- Log a warning to `system_logs` (`type: "caption_similarity_regen"`).
- Call the model **once** more with an extra suffix: "REGENERATION REQUIRED: previous draft was too similar to your last post. Use a different opening hook, sentence rhythm, and CTA."
- Use the regenerated caption if it scores `< 0.80`; otherwise keep it but log the failure. Never loop more than once (mirrors the existing one-shot regeneration pattern already used for foreign-landmark validation).

## 4. UI clarity — slot badge in Recent Posts

In `src/components/PostHistoryList.tsx`:

- Derive slot from the existing `[auto:slot]` / `[manual:slot]` marker already parsed elsewhere (or from `post_history.slot` if present — confirm column; if absent, regex the caption's `📍 ... · <Slot> Update` line).
- Render a small badge next to the platform chip: `Morning` / `Afternoon` / `Evening` with distinct color (amber / sky / indigo using existing semantic tokens). Adhoc/manual → neutral "Manual" chip.
- Pure presentation; no API changes.
